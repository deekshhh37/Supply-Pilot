# Technical Implementation Requirements (Mandatory)

## Overall Architecture

The project must remain a **single Next.js application**.

Do **not** introduce a separate Python backend, FastAPI service, or microservices.

Everything should run through **Next.js API Routes (TypeScript)** and be executable with:

```bash
npm install
npm run dev
```

The entire demo should work locally with a single command.

# Planner Agent

Implement a real Planner Agent.

The planner should:

* Understand the incoming business scenario
* Detect the problem type
* Select only the relevant specialized agents
* Execute them in sequence
* Aggregate results
* Produce a reasoning trace showing why each agent was invoked

The planner must not always invoke every agent.

Example:

Vendor delay

↓

Planner

↓

Inventory Agent

↓

Procurement Agent

↓

Customer Impact Agent

↓

Recommendation Agent

NOT

Planner

↓

Invoke every agent regardless of scenario

---

# Reasoning Engine

Implement a **deterministic, rule-based multi-factor reasoning engine**.

Recommendations should **not** be canned responses.

Every recommendation must be computed using real business signals.

Confidence should be calculated dynamically as a weighted function.

Example signals:

* Inventory days of supply
* Supplier lead time
* Customer SLA priority
* Delivery delay
* Weather severity
* Supplier reliability
* Procurement cost delta
* Stockout risk
* Historical vendor performance

Example formula:

Confidence =
0.25 × Inventory Health +
0.20 × SLA Urgency +
0.20 × Supplier Reliability +
0.15 × Lead Time +
0.10 × Weather Risk +
0.10 × Cost Impact

Normalize to 0–100%.

Display every contributing factor in the UI.

# Retrieval-Augmented Generation (RAG)

Implement a **real retrieval pipeline**.

Do NOT fake retrieval.

Use:

* In-process TF-IDF
* Cosine Similarity
* Markdown knowledge base
* Chunked documents

Knowledge base should include:

* Procurement Policies
* Vendor SLA
* Shipping SOP
* Warehouse Guidelines
* Escalation Procedures
* Customer Priority Matrix

Requirements:

* Chunk documents
* Compute TF-IDF vectors
* Calculate cosine similarity
* Retrieve Top-K chunks
* Show retrieval scores

Display retrieved evidence.

Example:

Evidence

Vendor SLA.md

Similarity: 0.93

Shipping SOP.md

Similarity: 0.89

Warehouse Policy.md

Similarity: 0.84
